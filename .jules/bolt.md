## 2024-05-24 - Requests Session Connection Pooling
**Learning:** In applications that make frequent HTTP requests to the same host (like sending multiple notifications to `ntfy.sh`), using a global `requests.Session()` object instead of `requests.put()`/`requests.post()` directly provides a massive performance boost. This is because `requests.Session()` uses `urllib3`'s connection pooling under the hood, allowing it to reuse the underlying TCP connection and avoid the overhead of repeated DNS lookups, TCP handshakes, and especially TLS negotiations for HTTPS endpoints.
**Action:** Always check if repeated HTTP calls are being made to the same host. If so, replace direct `requests.<method>` calls with a single shared `requests.Session()` instance to enable connection pooling.
